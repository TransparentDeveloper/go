# Go 동시성

### 1. 고루틴 (Goroutine)

- **정체**: Go 런타임이 관리하는 **초경량 스레드**.
- **특징**:
  - 함수 호출 앞에 `go` 키워드만 붙이면 즉시 비동기 실행.
  - OS 스레드보다 메모리 사용량이 압도적으로 적음 (약 2KB).
  - 메인 고루틴이 종료되면 모든 서브 고루틴도 강제 종료됨.

### 2. 채널 (Channel)

- **정체**: 고루틴 간에 데이터를 안전하게 주고받는 **통로(파이프라인)**.
- **특징**:
  - **공유 메모리 대신 통신**: 데이터를 복사해서 전달하여 동기화 문제를 방지.
  - **블로킹(Blocking)**: 데이터를 보내거나 받을 때 상대방이 준비될 때까지 대기함 (자동 동기화).
- **문법**:
  - 생성: `ch := make(chan 타입)`
  - 송신: `ch <- 데이터`
  - 수신: `변수 := <-ch`

### 3. 주요 포인트

1. **비동기 실행**: `go` 키워드는 호출 즉시 다음 코드로 넘어감.
2. **동기화**: 별도의 `Lock` 없이도 채널을 통해 고루틴 간의 작업 순서를 제어할 수 있음.
3. **데드락(Deadlock)**: 데이터를 보냈는데 받는 쪽이 없으면 프로그램이 멈추므로 주의.

### 4. 요약 비교

| 구분            | 고루틴 (Goroutine)      | 채널 (Channel)                         |
| :-------------- | :---------------------- | :------------------------------------- |
| **역할**        | 작업의 단위 (동시 실행) | 작업 간의 통신 (데이터 전달)           |
| **비유**        | 각자 일하는 일꾼        | 일꾼들이 물건을 주고받는 컨베이어 벨트 |
| **핵심 키워드** | `go`                    | `chan`, `<-`                           |
