# Go 인터페이스 (Interface)

### 1. 정체

- **행위(Method)의 집합**.
- "무엇을 가지고 있는가(Data)"가 아니라 **"무엇을 할 수 있는가(Behavior)"**에 집중함.

### 2. 특징: Duck Typing

- `implements` 같은 명시적 키워드가 없음.
- 구조체가 인터페이스에 정의된 메서드를 모두 가지고 있다면 **자동으로 구현한 것으로 간주**함.
  > "오리처럼 걷고 오리처럼 짖으면, 그것은 오리다."

### 3. 빈 인터페이스 (`interface{}` 또는 `any`)

- 정의된 메서드가 하나도 없는 인터페이스.
- **모든 타입**은 최소 0개 이상의 메서드를 가지므로, 어떤 값이라도 담을 수 있는 **만능 타입**이 됨.

### 4. 주요 포인트

1. **다중 구현**: 하나의 구조체가 여러 개의 인터페이스를 동시에 만족할 수 있음.
2. **작은 단위**: Go 마스터들은 인터페이스를 작게 만듦 (보통 메서드 1~2개).
   - 예: `Reader`, `Writer`, `Stringer`
3. **유연성**: 함수 인자로 인터페이스를 받으면, 특정 구조체에 종속되지 않는 유연한 코드를 짤 수 있음.
