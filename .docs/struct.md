# 📦 Go 구조체

### 1. 정체

- 서로 다른 타입의 데이터를 하나로 묶은 **사용자 정의 데이터 타입**.
- Go에는 `class`가 없으므로, 구조체가 데이터 설계도의 역할을 대신함.

### 2. 기본 문법

```go
type 구조체이름 struct {
    필드명 타입
}
```

### 3. 메서드와 리시버 (핵심!)

구조체에 기능을 붙일 때는 **리시버(Receiver)**를 사용함.

- **값 리시버** `(p Player)`: 데이터를 복사해서 사용. 원본 수정 불가.
- **포인터 리시버** `(p *Player)`: 데이터의 주소를 참조. **원본 수정 가능**.

### 4. 주요 포인트

1. **상속 없음**: 대신 구조체 안에 다른 구조체를 넣는 '임베딩(Embedding)' 방식을 사용함.
2. **대문자 규칙**: 필드명이 대문자로 시작하면 외부 패키지에서 접근 가능(Public), 소문자면 내부에서만 접근 가능(Private).
3. **생성자 없음**: 보통 `NewPlayer()` 같은 함수를 직접 만들어 객체 생성을 처리함.

### 5. 요약 비교

| 구분     | 일반 함수   | 구조체 메서드          |
| -------- | ----------- | ---------------------- |
| **형태** | `func Do()` | `func (p Player) Do()` |
| **호출** | `Do(p1)`    | `p1.Do()`              |
| **소속** | 독립적      | 특정 구조체에 종속     |
